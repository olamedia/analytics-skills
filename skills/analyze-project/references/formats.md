# Artifact Formats: analyze-project

Format definitions for the `analyze-project` skill. Defines the YAML frontmatter schema and markdown templates for output files.

## Frontmatter Schema

Every file generated by `analyze-project` starts with YAML frontmatter:

```yaml
---
generated-by: analyze-project
date: 2024-03-14T12:00:00Z
commit: abc1234
---
```

| Field | Type | Description |
|-------|------|-------------|
| `generated-by` | string | Always `analyze-project`. Identifies skill-owned files. |
| `date` | ISO 8601 | When the file was last generated/updated. |
| `commit` | string | Short git commit hash (`git rev-parse --short HEAD`) at time of generation. Used for staleness detection. |

## Output Files

| File | Path | Description |
|------|------|-------------|
| TechStack | `docs/TechStack.md` | Concise tech stack reference card |
| ProjectStructure | `docs/ProjectStructure.md` | Operational project map |

In monorepos, each service also gets:
- `[service-path]/docs/TechStack.md`
- `[service-path]/docs/ProjectStructure.md`
